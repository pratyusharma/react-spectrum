## Content

Buttons must have a visible label, and can optionally have an icon. Text only buttons accept a string
as children. Icons can also be added as children, with a sibling `<Text>` element for the label.

```tsx render expanded
"use client";
import {Button, Text} from '@react-spectrum/s2';
import Bell from '@react-spectrum/s2/icons/Bell';

<Button variant="secondary">
  <Bell />
  <Text>Icon + Label</Text>
</Button>
```

### Accessibility

If no visible label is provided (e.g. an icon only button),
an alternative text label must be provided to identify the control for accessibility. This should be added using
the `aria-label` prop.

### Internationalization

In order to internationalize a button, a localized string should be passed to the `children` or `aria-label` prop.

## Events

Buttons support user interactions via mouse, keyboard, and touch. You can handle all of these via the `onPress`
prop.

The following example uses an `onPress` handler to update a counter stored in React state.

```tsx render hideImports
"use client";
import {Button} from '@react-spectrum/s2';
import {useState} from 'react';

function Example() {
  let [count, setCount] = useState(0);

  return (
    <Button variant="primary" onPress={() => setCount(c => c + 1)}>{`${count} Dogs`}</Button>
  );
}
```

## Pending

Buttons can indicate that a quick progress task is taking place (e.g., saving settings on a server). After a 1 second delay, a loading indicator will be displayed in place of the button label and icon. You can trigger this behavior with the `isPending` prop. Button events are disabled while `isPending` is true.

### Loading Styles

Buttons support two loading animation styles:

- **Spinner** (default): An indeterminate spinner animation
- **Terminal**: A nostalgic terminal-style text animation that cycles through dots

```tsx render
"use client";
import {Button} from '@react-spectrum/s2';
import {useState} from 'react';

function Example() {
  let [isSpinnerLoading, setIsSpinnerLoading] = useState(false);
  let [isTerminalLoading, setIsTerminalLoading] = useState(false);

  let handleSpinnerPress = () => {
    setIsSpinnerLoading(true);
    setTimeout(() => setIsSpinnerLoading(false), 3000);
  };

  let handleTerminalPress = () => {
    setIsTerminalLoading(true);
    setTimeout(() => setIsTerminalLoading(false), 3000);
  };

  return (
    <div style={{display: 'flex', gap: 12, flexDirection: 'column'}}>
      <Button variant="primary" isPending={isSpinnerLoading} onPress={handleSpinnerPress}>
        Spinner Loading
      </Button>
      <Button 
        variant="secondary" 
        isPending={isTerminalLoading} 
        loadingStyle="terminal"
        onPress={handleTerminalPress}
      >
        Terminal Loading
      </Button>
    </div>
  );
}
```

### Terminal Loading Customization

The terminal loading style can be customized with additional props:

```tsx render
"use client";
import {Button} from '@react-spectrum/s2';
import {useState} from 'react';

function CustomTerminalExample() {
  let [isLoading, setIsLoading] = useState(false);

  let handlePress = () => {
    setIsLoading(true);
    setTimeout(() => setIsLoading(false), 4000);
  };

  return (
    <div style={{display: 'flex', gap: 12, flexDirection: 'column'}}>
      <Button 
        variant="primary" 
        isPending={isLoading} 
        loadingStyle="terminal"
        terminalText="Processing"
        terminalMaxDots={6}
        terminalSpeed={300}
        onPress={handlePress}
      >
        Custom Terminal
      </Button>
    </div>
  );
}
```

### Accessibility

Both loading styles provide proper accessibility support:

- Screen readers announce loading state changes
- Respects `prefers-reduced-motion` settings
- Maintains focus management during loading states
- Provides appropriate ARIA live announcements
